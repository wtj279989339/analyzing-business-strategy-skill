# BCG Deliverable Patterns

Use these patterns to guide deliverable structure, headline style, data presentation, and narrative flow.

## Slide Headline Style

BCG headlines state a conclusion or insight, not a topic. The headline alone should tell the story
if someone only reads titles.

**Do this (action-oriented):**
- "[Client] has been facing critical moment"
- "Transformation success relies on structured use of 6 levers"
- "Stable market outlook and [Client]'s solid positioning enables solid base case in EU/NA"
- "4 key initiatives are critical to address our local priorities"
- "Target market is a $2B globally, essentially flat for 20+ years"
- "Retailers hold most of the power in the industry"
- "[客户]在医院潜力评估方面精细化管理不足，导致整体销售效能较低"

**Don't do this (topic-only):**
- "Market Overview"
- "Competitive Analysis"
- "Financial Summary"
- "Recommendations"

Exception: section divider slides and agenda slides can be topic-only.

## Narrative Structure

### Standard Strategy / Market Entry
```
1. Cover (project name, date, client)
2. Agenda
3. Basis of Perspectives (sources, interviews, surveys)
4. Executive Summary (lead with the answer)
5. Context / Market Overview
   - Market size, growth, segmentation
   - Key trends (3-5 macro trends)
6. Competitive Landscape
   - Positioning maps
   - Benchmarking tables
7. Deep-dive Sections (3-5 storylines)
   - Each: Headline insight → Data → "Implications to [client]"
8. Financial Impact / Business Case
9. Recommendations with Prioritization
10. Roadmap / Implementation Plan
11. Appendix (methodology, additional data)
```

### Due Diligence (CDD)
```
1. Cover
2. Project Approach and Workplan (modules, timeline)
3. Basis of Perspectives (interview count, survey status, secondary sources)
4. Summary of Findings (by module)
5. Module Deep-dives:
   - Market outlook
   - Brand assessment
   - Channel analysis
   - Consumer survey results
   - Value creation opportunities
6. Key Issues Analysis
7. Path Forward
8. Appendix
```

Readouts are progressive: Readout #1 (outside-in market), #2 (company performance),
#3 (value creation + final view).

### Turnaround / Transformation
```
1. Cover
2. Project Agenda (timeline of working groups and steering committees)
3. Current State ("From...")
   - Revenue/profit decline visualization
   - Root cause diagnosis (5-7 key issues)
4. Transformation Levers (6-8 levers, each with clear label)
   - Each lever: Issue → Solution → Financial impact
5. Coordinated Lever Impact (waterfall showing combined P&L effect)
6. Detailed Lever Plans
7. Financial Impact Summary (P&L bridge: current → target)
8. Implementation Roadmap (phased, with milestones)
```

## "Basis of Perspectives" Slide

BCG always shows their work upfront. This builds credibility before any analysis.

```
Basis of our perspectives:
- 96 interviews conducted
  - 70 competitors and experts (by region)
  - 26 retailers (by region)
- Consumer survey: N=3,277 across 3 geographies
- Secondary research: Euromonitor, SIA, Capital IQ, company filings
- 25 mystery shopping calls
```

Include this as a dedicated slide early in the deck. List interview counts, survey sample sizes,
and named secondary sources. The more specific, the more credible.

## Data Presentation Patterns

### Market Sizing
- Always show: total market size, CAGR, segmentation
- Use ranges when uncertain: "$590M-$650M"
- Show historical + forecast with clear divider
- Label CAGR prominently: "'23-'30 CAGR: +4%"
- Break down by geography, segment, channel

### Benchmarking
- Side-by-side comparison tables with named competitors
- "Implications to [client]" box after each benchmark
- Use specific competitor examples: "Uniqlo has self-developed a big-data enabled tool..."

### Financial Impact
- P&L waterfall: current state → lever impacts → target state
- Each lever shows specific RMB/$ impact
- Operating profit ratio shown as target: "2028 Operating profit ratio: 7.0%"

### Driver Trees
- Visual decomposition: Revenue = Population × Incidence × Avg Spend × Uplift
- Color-code: inputs/assumptions vs. calculated values vs. repeated elements
- "Output First" approach: define KPIs → draft output → work back to inputs

### Trend Visualization
- "From... To..." framing for transformation
- Current state (left) → Future state (right) with arrows
- Color-coded growth indicators: green (>2.5%), yellow (0-2.5%), red (<0%)

## Chinese Deck Conventions

Same structural rigor as English decks. Key differences:
- Section headers often use "| " separator: "AI医院潜力测算| 讨论点"
- Transformation timelines use phases: "第一阶段：快速提升" → "第二阶段：推动转型"
- "痛点" (pain points) → "行业领先做法" (industry best practices) → solution
- Footnotes and sources in Chinese with original data source names

## Key Patterns to Replicate

1. **Lead with the answer.** Executive summary first, then evidence. Never build up to a reveal.

2. **Every section ends with "Implications to [client]."** Don't just present data — tell the
   client what it means for them specifically.

3. **Show your sources early.** The "Basis of Perspectives" slide is not optional. It's how BCG
   establishes credibility before making any claims.

4. **Use specific numbers, not vague language.** Not "significant growth" but "15% CAGR from
   '23 to '30." Not "many interviews" but "96 interviews across 3 regions."

5. **Benchmark against named competitors.** Not "industry peers" but "Uniqlo, Heilan, Nike."
   Show what they do differently and why it matters.

6. **Frame transformation as levers.** Number them (Lever 1-6), give each a clear label, show
   individual and combined financial impact.

7. **Use "From... To..." for change narratives.** Current state on left, future state on right.
   Makes the gap and the solution visually obvious.

8. **Progressive depth in readouts.** Don't dump everything at once. Readout #1 = market context,
   #2 = company assessment, #3 = value creation. Each builds on the last.

9. **Ranges over false precision.** "$590M-$650M" is more credible than "$623M." Use ranges
   for estimates, precise numbers only when sourced.

10. **Footnotes are mandatory.** Every data slide has source attribution at the bottom. Every
    assumption is noted. Every abbreviation is defined on first use.
